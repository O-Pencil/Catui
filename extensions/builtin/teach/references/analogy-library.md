# Analogy Library

A curated collection of analogies mapping complex concepts to familiar mental models.
Each analogy is tested for accuracy and memorability.

---

## Usage Notes

- Pick **ONE** analogy per concept per session — don't mix metaphors
- If a learner responds with confusion, switch to an alternative analogy
- Analogies are **entry points**, not endpoints — always follow up with the real mechanism
- Mark analogies as 🟡 (rough match) vs 🟢 (very close match) when the mapping isn't perfect
- When an analogy breaks down, say so honestly: "This analogy doesn't hold up here, actually..."
- **CRITICAL**: All analogies must be culturally appropriate and universally understandable

---

## Adding New Analogies

When creating new analogies:

1. **Test for accuracy**: Does the analogy actually map correctly?
2. **Test for memorability**: Will learners remember it?
3. **Test for universality**: Does it work across cultures?
4. **Note limitations**: Where does the analogy break down?
5. **Provide alternatives**: Have backup analogies ready

## Example Usage

**Teaching about variables**:
> "Think of a variable like a labeled box. You can put things in it, take things out, and change what's inside. The label (variable name) helps you find it later."

**Teaching about functions**:
> "A function is like a recipe. You give it ingredients (parameters), it follows steps (code), and gives you a dish (return value). You can use the same recipe many times."

**Teaching about loops**:
> "A loop is like washing dishes. You pick up a dish, wash it, dry it, put it away, then repeat for the next dish until you're done."
